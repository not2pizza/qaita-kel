# Установка Antigravity Coffee на iPad (нативное приложение)

Сборка нативного `.ipa`/запуск на устройстве делается **на Mac с Xcode**.
Проект под капотом — Capacitor (`appId: com.antigravity.coffee`, веб-сборка из `dist/`).

> **Зависимости iOS — через Swift Package Manager (SPM), НЕ CocoaPods.**
> CocoaPods устарел; этот проект уже на SPM (`ios/App/CapApp-SPM/`, открывается
> `App.xcodeproj` без `.xcworkspace`). Ставить CocoaPods не нужно — пакеты тянет
> сам Xcode.

---

## 0. Что нужно на Mac (один раз)

- **macOS + Xcode** (у тебя есть ✅). Открой Xcode хотя бы раз, прими лицензию.
- **Node.js 20+**: `node -v` (если нет — https://nodejs.org или `brew install node`)
- **Apple ID** для подписи (бесплатного хватит для теста; для киоска «навсегда» — платный Apple Developer, см. §7).
- Командные инструменты Xcode: `xcode-select --install`
- CocoaPods **не нужен** — зависимости резолвит Xcode через SPM.

---

## 1. Забрать код на Mac

```bash
git clone https://github.com/not2pizza/qaita-kel.git
cd qaita-kel
```

## 2. Создать `.env` (КРИТИЧНО)

`.env` не лежит в git (там ключи). Без него приложение не достучится до базы.
Создай файл `.env` в корне проекта:

```bash
cat > .env <<'EOF'
VITE_SUPABASE_URL=https://ТВОЙ-ПРОЕКТ.supabase.co
VITE_SUPABASE_ANON_KEY=твой-anon-ключ
EOF
```

> Значения возьми из Supabase → Project Settings → API. Это те же ключи, что в `.env` на Windows.

## 3. Установить зависимости

```bash
npm install
```

## 4. Собрать веб + синхронизировать в iOS

```bash
npm run ios:sync
```
Эта команда: `vite build` → копирует `dist/` в нативный проект → обновляет
зависимости Capacitor в SPM-пакете. (CocoaPods/`pod install` не вызывается.)

## 5. Открыть в Xcode

```bash
npm run ios:open
```
Откроется **`ios/App/App.xcodeproj`** (именно `.xcodeproj`, не `.xcworkspace` —
проект на SPM). При первом открытии Xcode сам подтянет Swift-пакеты
(статус «Resolving Package Versions…» внизу). Если не началось:
**File → Packages → Resolve Package Versions**.

---

## 6. Настроить подпись и запустить на iPad

1. В Xcode слева выбери проект **App** → таргет **App** → вкладка **Signing & Capabilities**.
2. Поставь галочку **Automatically manage signing**.
3. **Team** → выбери свой Apple ID (Add an Account…, если ещё не добавлен).
4. Если ругается на Bundle Identifier «не уникален» — поменяй `com.antigravity.coffee` на свой, например `com.tvoyakompaniya.antigravity` (только в Xcode-поле; в `capacitor.config.ts` менять не обязательно для запуска).
5. Подключи iPad кабелем. На iPad: «Доверять этому компьютеру».
6. Вверху Xcode выбери своё устройство (вместо симулятора).
7. Нажми ▶️ **Run** (Cmd+R). Первая сборка идёт дольше.

### На iPad после установки
- Если приложение не открывается с ошибкой «Untrusted Developer»:
  **Настройки → Основные → VPN и управление устройством → [твой Apple ID] → Доверять**.
- При первом запуске согласись на **доступ к камере** (нужно для распознавания лиц).
- Держи iPad **вертикально** — приложение залочено в портрет.

---

## 7. Бесплатный vs платный Apple ID

- **Бесплатный Apple ID**: приложение работает **7 дней**, потом надо пересобрать/переустановить. Ок для теста.
- **Apple Developer Program ($99/год)**: подпись на **1 год**, можно ставить «по воздуху» и публиковать. **Рекомендуется для боевого киоска.**

---

## 8. Режим киоска (важно для точки)

Чтобы гость не вышел из приложения и не лазил по iPad:

**Guided Access (Гид-доступ):**
1. Настройки → Универсальный доступ → **Гид-доступ** → включить.
2. Задать код-пароль для Гид-доступа.
3. Запустить приложение → **тройное нажатие** боковой кнопки → Старт.
4. Выход: тройное нажатие + код.

Дополнительно для стойки:
- Настройки → Экран и яркость → **Автоблокировка → Никогда**.
- Настройки → Аккумулятор — держать на зарядке.
- Скрытый вход в админку в самом приложении: **удержать логотип ☕ ~1 сек**.

---

## 9. Перед открытием — данные в Supabase

В SQL-редакторе Supabase прогнать миграции (если ещё не):
- `supabase/migrations/0002_brand_settings.sql` — брендинг (название/цвета/лого).
- `supabase/migrations/0003_kiosks.sql` — таблица киосков.

Затем при желании заполнить:
- товары, филиалы (`branches` — название + адрес показываются на экране),
- модификаторы (`modifier_groups` / `modifier_options` / `product_modifier_groups`) — иначе работает запасной набор Size/Milk/Syrup,
- привязать киоск к филиалу через админку (☕ удержать → карточка «This kiosk»).

---

## 10. Обновление приложения после изменений кода

```bash
git pull
npm install        # если менялись зависимости
npm run ios:sync
npm run ios:open   # → Run в Xcode
```

## Частые проблемы

| Симптом | Причина / решение |
|---|---|
| Белый экран, нет товаров | Нет/неверный `.env` (§2) или не прогнаны миграции (§9) |
| «Untrusted Developer» | Доверить профиль в Настройках (§6) |
| Камера не работает | Не выдан доступ; Настройки → приложение → Камера |
| Ошибки Swift-пакетов / «Missing package» | File → Packages → **Reset Package Caches**, затем **Resolve Package Versions** |
| Приложение «протухло» через неделю | Бесплатный Apple ID — нужен платный (§7) |
| Перевёрнут/растянут | Должно быть портретно; держать iPad вертикально |
